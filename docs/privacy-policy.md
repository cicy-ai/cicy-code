# cicy-code 数据透明与隐私说明

> 版本: 1.0 | 生效日期: 2026-03-11

---

## 我们做了什么

cicy-code 为您提供基于 Web 的 AI 编程助手管理平台。当您使用平台上的 kiro-cli 工作时，您的 HTTP 流量将通过我们的代理服务（mitmproxy），用于：

1. **对话可视化** — 将 CLI 的纯文本交互转化为可视化的对话界面
2. **用量监控** — 展示您的 Credit 消耗、模型使用情况
3. **流量统计** — 请求次数、数据量等基础指标

## 我们记录哪些数据

| 数据类型 | 记录 | 展示 | 说明 |
|----------|------|------|------|
| 请求 URL 和方法 | ✅ | ✅ | 如 POST q.us-east-1.amazonaws.com |
| 对话内容 (Q&A) | ✅ | ✅ | 您发送的消息和 AI 的回复 |
| Credit 消耗 | ✅ | ✅ | 每轮对话的 Credit 用量 |
| 工具调用记录 | ✅ | ✅ | fs_read、execute_bash 等 |
| 请求/响应大小 | ✅ | ✅ | KB 级别的流量统计 |
| 认证凭证 (Token) | ✅ 记录 | ❌ 脱敏 | Bearer Token、AWS 密钥等不会明文展示 |
| 性能指标 | ✅ | ✅ | 首 token 延迟、响应时间 |

## 我们不会做的事

- ❌ **不会分享** — 您的数据不会分享给其他用户或第三方
- ❌ **不会传播** — 数据不会用于训练、分析或任何商业用途
- ❌ **不会使用您的凭证** — 记录的 AWS Token 仅用于日志存储，绝不会用于调用任何 API
- ❌ **不会跨租户访问** — 每个用户只能看到自己的数据，严格按 pane_id 隔离

## 数据安全措施

- **租户隔离**: 数据按 `pane_id` 严格隔离，用户只能访问自己的记录
- **凭证脱敏**: 前端展示时，认证相关的 Header（Authorization、Security Token 等）自动脱敏处理
- **传输加密**: 平台通过 HTTPS (Cloudflare Tunnel) 提供访问
- **自动清理**: 数据保留 30 天后自动清理
- **可选退出**: 您可以随时关闭代理功能（设置 `proxy.enable = false`），关闭后流量不经过代理，不产生任何记录

## 技术实现

```
您的 kiro-cli → mitmproxy (代理) → AWS Q Developer API
                    │
                    ▼
              Redis (缓冲) → MySQL (存储) → Web UI (仅您可见)
```

- 代理服务运行在平台服务器上，使用标准的 mitmproxy 开源组件
- 数据存储在平台的 MySQL 数据库中，按租户隔离
- 您在 Web UI 上看到的就是您自己的 CLI 交互记录的可视化呈现

## 您的权利

- **查看**: 随时在 Web UI 查看自己的完整流量记录
- **删除**: 可以请求删除自己的所有历史数据
- **退出**: 随时关闭代理，恢复直连模式
- **导出**: 可以导出自己的对话记录
